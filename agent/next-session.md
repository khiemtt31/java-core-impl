## Session Goal

By the end of this session, you should be able to explain:

- What `static` means for variables, methods, blocks, and nested classes
- How `static` differs from instance members
- What `final` means for variables, methods, and classes
- The difference between `final` and immutability

## Self-Test Questions

### `static`

- When is a static variable initialized?
- Can a static method access instance variables?
- Why is `main()` static?
- What is the difference between a static block and a constructor?

### `final`

- What can be made `final` in Java?
- What is the difference between a final object and an immutable object?
- Why is `String` immutable?
- What happens if you try to reassign a final variable?

### `this` and `super`

- When should you use `this()` versus `super()`?
- Can both be called in the same constructor?
- How does constructor chaining work?
- What is the role of `super` in method overriding?

## Practice Tasks

- Write a class with a `static` counter shared by all objects
- Write a class that uses a `final` field initialized in the constructor
- Create a small inheritance example that uses `this` and `super`
- Explain each example in interview language

## Exit Criteria

You are ready to move on when you can answer the self-test questions without notes and explain the examples clearly in your own words.
